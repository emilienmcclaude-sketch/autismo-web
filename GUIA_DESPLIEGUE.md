# Entendamos el Autismo — Guía de despliegue

Esta guía asume que ya tenés cuentas creadas en: GitHub, Google AI Studio (con tu clave de Gemini guardada), Vercel y Supabase.

---

## PASO 1 — Crear la tabla en Supabase

1. Entrá a tu proyecto en supabase.com
2. En el menú lateral, hacé clic en **"SQL Editor"**
3. Pegá y ejecutá ("Run") este código:

```sql
create table eventos (
  id bigint generated by default as identity primary key,
  tipo text not null,
  contenido text not null,
  creado_en timestamp with time zone default now()
);
```

Esto crea la tabla donde se van a guardar todas las búsquedas y clics.

---

## PASO 2 — Conseguir las claves de Supabase

1. En tu proyecto de Supabase, ir a **"Project Settings" → "API"**
2. Vas a ver dos datos importantes:
   - **Project URL** (algo como `https://xxxxx.supabase.co`)
   - **service_role key** (una clave larga, en la sección "Project API keys")
3. Copiá ambos y guardalos junto a tu clave de Gemini

⚠️ La "service_role key" tiene permisos totales sobre tu base de datos — nunca la subas a GitHub ni la pongas en código visible al público. Por eso en este proyecto solo se usa del lado del servidor.

---

## PASO 3 — Subir el proyecto a GitHub

1. Entrá a github.com y hacé clic en **"New repository"**
2. Poné un nombre, por ejemplo `autismo-web`, dejalo en "Public" o "Private" (cualquiera funciona)
3. Creá el repositorio (sin agregar README ni .gitignore, ya los tenemos)
4. En tu computadora, descomprimí la carpeta del proyecto que te compartí
5. Seguí las instrucciones que GitHub te muestra bajo **"…or push an existing repository from the command line"**, o si preferís sin usar comandos:
   - Hacé clic en **"uploading an existing file"** en la página del repositorio recién creado
   - Arrastrá todos los archivos y carpetas del proyecto (excepto si ves una carpeta `node_modules`, esa no se sube)
   - Confirmá la subida ("Commit changes")

---

## PASO 4 — Conectar el repositorio a Vercel

1. Entrá a vercel.com → **"Add New" → "Project"**
2. Elegí el repositorio `autismo-web` que acabás de crear en GitHub
3. Antes de hacer clic en "Deploy", buscá la sección **"Environment Variables"**
4. Agregá estas tres variables (los nombres deben ser EXACTOS):

| Nombre | Valor |
|---|---|
| `GEMINI_API_KEY` | tu clave de Gemini (la que empieza con AQ.Ab...) |
| `SUPABASE_URL` | la Project URL de Supabase |
| `SUPABASE_SERVICE_KEY` | la service_role key de Supabase |

5. Hacé clic en **"Deploy"**
6. Esperá unos minutos — Vercel va a darte una URL pública como `autismo-web.vercel.app`

¡Listo! Esa URL ya es tu web funcionando de verdad, accesible desde cualquier lugar.

---

## PASO 5 — Verificar que todo funciona

1. Abrí la URL que te dio Vercel
2. Probá escribir una pregunta o tocar un tema del índice
3. Volvé a Supabase → **"Table Editor" → "eventos"** — deberías ver la búsqueda registrada ahí

---

## Si algo falla

- **La IA no responde / da error**: revisá que `GEMINI_API_KEY` esté bien copiada en Vercel (sin espacios de más)
- **No se guardan los eventos en Supabase**: revisá `SUPABASE_URL` y `SUPABASE_SERVICE_KEY`, y que la tabla `eventos` exista
- Cualquier cambio en las variables de entorno en Vercel requiere un "Redeploy" (botón en el panel del proyecto) para que se aplique
